# Scenario 4: 온디바이스 스마트 이미지 갤러리 검색기 (Local Image Captioner & Gallery Search)

## 💡 개요
모바일 갤러리 내의 사진들을 멀티모달 능력을 지닌 Gemma 4 Vision(VLM) 모델을 이용하여 오프라인 로컬 환경에서 자동 분석(Captioning)합니다. 각 이미지의 세부 묘사와 분위기를 텍스트 캡션으로 기록하여 사용자가 구글 포토처럼 "바닷가에서 찍은 사진", "강아지가 뛰노는 모습"과 같은 자연어 쿼리로 사진을 쉽게 검색할 수 있게 만듭니다.

---

## 🛠️ 주요 기술 스펙
- **Gemma 4 Vision (Multimodal VLM)**: 이미지를 입력으로 받아 고품질의 자연어 설명을 생성하는 비전-언어 통합 온디바이스 추론 모델 연동.
- **로컬 텍스트 검색 및 임베딩 Indexing**: 생성된 설명문을 바탕으로 플러터 로컬 데이터베이스 내 키워드 검색 엔진 구축 또는 로컬 오프라인 임베딩 모델을 활용한 코사인 유사도 기반 벡터 검색(Vector Search).

---

## 🚀 아키텍처 및 데이터 흐름
```text
모바일 갤러리 이미지 선택 
  ➡️ 로컬 Gemma 4 Vision 추론
  ➡️ 자연어 캡션 생성: "화창한 날 해변에서 모래성을 쌓고 있는 어린아이와 배경의 파도"
  ➡️ 캡션 텍스트 로컬 DB / Vector DB 저장
  ➡️ 사용자 검색어 입력: "바다 놀이터"
  ➡️ 벡터 유사도 / 키워드 매칭 수행
  ➡️ 매칭된 이미지 갤러리 카드 그리드 시각화
```

---

## 📅 로드맵 및 도전 과제
1. **배경 백그라운드 프로세싱**: 모바일 디바이스가 유휴 상태(Idle)이거나 충전 중일 때 백그라운드 서비스(Background Worker)를 통해 갤러리 신규 이미지를 조용히 배치 캡셔닝하는 기법 필요.
2. **Vision 모델 크기 최적화**: 모바일 비전 모델의 거대한 크기를 극복하기 위해 경량 모바일 비전 아키텍처(예: Paligemma 계열) 최적 포팅 연구.
